{{ ui.help_label("Closed Candle Discipline", "closed_candle") }}

CMVA may display a current forming candle, but only closed candles enter storage, features, models, shock labels, regimes, diagnostics, and validation.

{{ ui.help_label("Data Accumulation", "bootstrap_progress") }}

Historical Binance Spot candles are accumulated by symbol and interval, then checked for missing gaps, duplicate timestamps, OHLC consistency, volume validity, and unclosed candle leakage.

{{ ui.help_label("Volatility", "volatility") }}

Realized volatility, EWMA volatility, range volatility, and GARCH forecast volatility describe return variability over the selected candle interval.

{{ ui.help_label("GARCH Forecast", "garch_forecast") }}

The forecast is sigma_{t+1|t}, the next-candle volatility forecast from information available at time t. It is not a price direction forecast.

{{ ui.help_label("Trend", "trend_slope") }}

Trend features use rolling OLS on log price and diagnostic t-statistics. They describe the observed path and are not buy or sell instructions.

{{ ui.help_label("Correlation", "average_correlation") }}

Correlation, BTC beta, PCA1 share, and dispersion describe whether the market is moving together or separating by symbol.

{{ ui.help_label("Shock Score", "shock_score") }}

Shock scoring compares the return observed at t with the forecast made before t.

{{ ui.help_label("Walk-Forward Validation", "walk_forward") }}

Backtesting in CMVA means model validation: forecast losses, calibration, and residual tests.

{{ ui.help_label("Look-Ahead Bias", "lookahead_bias") }}

Features at t use data through t; forecast evaluation aligns forecast_{t|t-1} against r_t.
